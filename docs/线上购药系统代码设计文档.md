## 1 概述

本文档是线上购药系统的代码设计说明书，选取了系统中两个核心模块进行详细的代码级设计：智能药师咨询模块和处方管理模块。

智能药师咨询模块是系统的核心差异化功能，基于大语言模型提供7x24小时智能用药咨询服务，包含AI对话、症状分析、风险分级、人工转接等复杂业务逻辑。处方管理模块是保障处方药合规销售的关键环节，涉及处方上传、OCR识别、AI配伍检查、药师人工审核等多个步骤。

代码设计采用Java语言，基于Spring Boot框架，遵循分层架构原则（Controller-Service-Repository），每个类的设计包含：类职责说明、方法定义（含**入参**/**出参**/**返回值**）、方法功能描述、方法内部逻辑（以注释形式描述）。

### 2.3 AIService（AI服务层）

**类职责**

封装与大语言模型的交互逻辑（基于 LangChain4j），负责调用 DeepSeek V4 生成用药咨询回复。集成 RAG 检索增强技术：通过 DeepSeek Embedding API 将用户问题转为 1536 维向量，再由 pgvector 余弦相似度（<=>）进行语义检索匹配药品（向量检索为主路径，keyword LIKE 仅作降级），最后将药品上下文与对话历史注入 ChatRequest，确保回复基于系统中实际存在的药品信息。

     * **逻辑**：
     *  1. 构建系统提示词（system prompt），定义AI药师角色和行为规范
     *  2. 将knowledgeDocs格式化为上下文文本
     *  3. 调用promptService.buildPrompt(userContent, context)组装完整提示
     *  4. 调用llmClient.chat(prompt)发送请求到LLM服务
     *  5. 解析LLM返回的文本内容

### 2.4 RiskAssessmentService（风险评估服务）

**类职责**

对AI生成的回复内容进行风险分级评估，识别潜在的用药安全风险，决定回复的展示策略。

#### `RiskAssessmentService.java`

```java
package com.ops.consult.service;

import org.springframework.stereotype.Service;

@Service
public class RiskAssessmentService {

    private final RiskKeywordRepository riskKeywordRepo;

    public RiskAssessmentService(RiskKeywordRepository riskKeywordRepo) {
        this.riskKeywordRepo = riskKeywordRepo;
    }

    /**
     * 评估AI回复的风险等级
     * **入参**：aiContent（String，AI生成的回复内容）
     * **出参**：RiskLevel（枚举：LOW/MEDIUM/HIGH）
     * **功能**：分析AI回复内容，判断其风险等级
     * **逻辑**：
     *  1. 加载高风险关键词库（如"确诊""一定""保证治愈"等绝对化表述）
     *  2. 加载中风险关键词库（如"建议""可能""处方药"等）
     *  3. 检查内容是否包含高风险关键词：
     *     - 如果包含：返回HIGH（需要转人工）
     *  4. 检查内容是否包含中风险关键词：
     *     - 如果包含：返回MEDIUM（需要药师复核）
     *  5. 检查内容是否涉及处方药推荐：
     *     - 如果是：返回MEDIUM
     *  6. 其他情况返回LOW（可直接展示）
     */
    public RiskLevel assess(String aiContent) {
        // 1. 加载关键词库
        List<String> highRiskKeywords = riskKeywordRepo
                .findKeywordsByLevel(RiskLevel.HIGH);
        List<String> mediumRiskKeywords = riskKeywordRepo
                .findKeywordsByLevel(RiskLevel.MEDIUM);
        // 2. 检查高风险
        for (String keyword : highRiskKeywords) {
            if (aiContent.contains(keyword)) {
                return RiskLevel.HIGH;
            }
        }
        // 3. 检查中风险
        for (String keyword : mediumRiskKeywords) {
            if (aiContent.contains(keyword)) {
                return RiskLevel.MEDIUM;
            }
        }
        // 4. 默认低风险
        return RiskLevel.LOW;
    }
}
```

### 2.5 PharmacyKnowledgeService（药学知识库服务）

**类职责**

实现RAG（检索增强生成）功能，从药学知识库中检索与用户问题相关的权威资料，为AI回复提供知识支撑。

#### `PharmacyKnowledgeService.java`

```java
package com.ops.consult.service;

import org.springframework.stereotype.Service;
import com.ops.consult.repository.KnowledgeRepository;

@Service
public class PharmacyKnowledgeService {

    private final KnowledgeRepository knowledgeRepo;
    private final EmbeddingService embeddingService;

    public PharmacyKnowledgeService(KnowledgeRepository knowledgeRepo,
            EmbeddingService embeddingService) {
        this.knowledgeRepo = knowledgeRepo;
        this.embeddingService = embeddingService;
    }

    /**
     * 检索药学知识
     * **入参**：query（String，用户输入的查询内容）
     * **出参**：List<KnowledgeDoc>（相关知识文档列表，最多5条）
     * **功能**：基于向量相似度从药学知识库中检索相关资料
     * **逻辑**：
     *  1. 调用embeddingService将query转换为向量表示
     *  2. 使用向量数据库进行相似度搜索（cosine similarity）
     *  3. 取相似度最高的前5条结果
     *  4. 如果最高相似度低于阈值0.7，返回空列表（避免引入不相关知识）
     *  5. 返回KnowledgeDoc列表
     */
    public List<KnowledgeDoc> search(String query) {
        // 1. query向量化
        float[] queryVector = embeddingService.embed(query);
        // 2. 向量相似度搜索
        List<KnowledgeDoc> results = knowledgeRepo
                .searchByVector(queryVector, 5, 0.7f);
        // 3. 返回结果
        return results;
    }
}
```

### 3.3 OCRService（OCR识别服务）

**类职责**

封装处方图片的OCR文字识别功能，调用第三方OCR引擎或自研模型，提取处方中的关键信息。

#### `OCRService.java`

```java
package com.ops.prescription.service;

import org.springframework.scheduling.annotation.Async;
import org.springframework.stereotype.Service;
import com.ops.prescription.client.OCRClient;

@Service
public class OCRService {

    private final OCRClient ocrClient;
    private final PrescriptionRepository prescriptionRepo;
    private final PrescriptionItemRepository itemRepo;
    private final DrugCheckService drugCheckService;

    public OCRService(OCRClient ocrClient,
            PrescriptionRepository prescriptionRepo,
            PrescriptionItemRepository itemRepo,
            DrugCheckService drugCheckService) {
        this.ocrClient = ocrClient;
        this.prescriptionRepo = prescriptionRepo;
        this.itemRepo = itemRepo;
        this.drugCheckService = drugCheckService;
    }

    /**
     * 异步识别处方图片（@Async异步方法）
     * **入参**：prescriptionId（Long，处方ID）、imageUrl（String，图片URL）
     * **出参**：无（异步执行）
     * **功能**：对处方图片进行OCR识别，提取处方信息，识别完成后自动触发AI检查
     * **逻辑**：
     *  1. 更新处方ocrStatus=1（识别中）
     *  2. 调用ocrClient.recognize(imageUrl)进行OCR识别
     *  3. 解析OCR结果，提取：医院名称、医生姓名、处方日期、药品列表
     *  4. 将解析结果保存到处方主表（hospitalName, doctorName等）
     *  5. 将药品明细保存到处方明细表
     *  6. 更新ocrStatus=2（识别成功）
     *  7. 自动触发drugCheckService.checkAsync(prescriptionId)进行AI配伍检查
     * **异常处理**：
     *  - 如果OCR识别失败：更新ocrStatus=3（识别失败），记录失败原因
     *  - 如果图片质量不佳：更新ocrStatus=3，提示用户重新上传
     */
    @Async("taskExecutor")
    public void recognizeAsync(Long prescriptionId, String imageUrl) {
        try {
            // 1. 更新状态为识别中
            Prescription entity = prescriptionRepo.findById(prescriptionId)
                    .orElseThrow();
            entity.setOcrStatus(1);
            prescriptionRepo.save(entity);
            // 2. 调用OCR识别
            OCRResult result = ocrClient.recognize(imageUrl);
            // 3. 解析并保存结果
            // ... 解析OCR结果，保存药品明细
            // 4. 更新状态为识别成功
            entity.setOcrStatus(2);
            prescriptionRepo.save(entity);
            // 5. 触发AI配伍检查
            drugCheckService.checkAsync(prescriptionId);
        } catch (Exception e) {
            // 异常处理：更新状态为识别失败
            Prescription entity = prescriptionRepo.findById(prescriptionId).orElse(null);
            if (entity != null) {
                entity.setOcrStatus(3);
                prescriptionRepo.save(entity);
            }
        }
    }
}

    private final PrescriptionAuditService auditService;

    public DrugCheckService(PrescriptionRepository prescriptionRepo,
            PrescriptionItemRepository itemRepo,
            DrugInteractionRepository interactionRepo,
            PrescriptionAuditService auditService) {
        this.prescriptionRepo = prescriptionRepo;
        this.itemRepo = itemRepo;
        this.interactionRepo = interactionRepo;
        this.auditService = auditService;
    }

    /**
     * 异步AI配伍检查（@Async异步方法）
     * **入参**：prescriptionId（Long，处方ID）
     * **出参**：无（异步执行）
     * **功能**：对处方中的药品进行配伍禁忌检查和剂量合理性分析
     * **逻辑**：
     *  1. 更新处方aiCheckStatus=1（检查中）
     *  2. 查询处方所有药品明细
     *  3. 逐一对每个药品进行以下检查：
     *     a. 配伍禁忌检查：查询drug_interaction表，
     *        检查该药品与处方中其他药品是否存在相互作用
     *     b. 剂量合理性检查：对比药品标准剂量范围
     *  4. 根据检查结果更新每个药品的ai_check_result：
     *     - 1通过：无禁忌，剂量合理
     *     - 2警告：存在轻微相互作用或剂量偏高
     *     - 3严重：存在严重配伍禁忌或剂量超标
     *  5. 汇总处方整体检查状态：
     *     - 如有任何严重项：aiCheckStatus=3（严重）
     *     - 如有警告项无严重项：aiCheckStatus=2（警告）
     *     - 全部通过：aiCheckStatus=1（通过）
     *  6. 更新处方pharmacistStatus=3（待药师审核）
     *  7. 自动提交审核任务给药师工作台
     */
    @Async("taskExecutor")
    public void checkAsync(Long prescriptionId) {
        // 1. 更新状态为检查中
        Prescription entity = prescriptionRepo.findById(prescriptionId).orElseThrow();
        entity.setAiCheckStatus(1);
        prescriptionRepo.save(entity);
        // 2. 查询药品明细
        List<PrescriptionItem> items = itemRepo.findByPrescriptionId(prescriptionId);
        // 3. 配伍禁忌检查 + 剂量检查
        // ... 检查逻辑
        // 4. 提交药师审核
        auditService.assignToPharmacist(prescriptionId);
    }
}
```

### 3.5 PrescriptionAuditService（处方审核服务）

**类职责**

管理药师工作队列，实现处方的分配和审核流程调度。确保每张处方都能被及时分配至在线药师进行审核。

#### `PrescriptionAuditService.java`

```java
package com.ops.prescription.service;

import org.springframework.stereotype.Service;
import java.util.concurrent.*;

@Service
public class PrescriptionAuditService {

    private final PharmacistRepository pharmacistRepo;
    private final PrescriptionRepository prescriptionRepo;
    private final LinkedBlockingQueue<Long> auditQueue;

    public PrescriptionAuditService(PharmacistRepository pharmacistRepo,
            PrescriptionRepository prescriptionRepo) {
        this.pharmacistRepo = pharmacistRepo;
        this.prescriptionRepo = prescriptionRepo;
        this.auditQueue = new LinkedBlockingQueue<>();
    }

    /**
     * 将处方分配至药师工作台
     * **入参**：prescriptionId（Long，处方ID）
     * **出参**：无
     * **功能**：将待审核处方加入药师审核队列，等待药师领取
     * **逻辑**：
     *  1. 校验处方状态为"待药师审核"
     *  2. 将prescriptionId加入审核队列（auditQueue.offer）
     *  3. 查找当前在线且空闲的药师
     *  4. 如果有空闲药师：
     *     - 从队列取出处方分配给该药师
     *     - 更新药师状态为"审核中"
     *     - 发送消息通知药师有新处方待审
     *  5. 如果没有空闲药师：
     *     - 处方留在队列中等待
     *     - 记录待审处方数量（用于负载监控）
     */
    public void assignToPharmacist(Long prescriptionId) {
        Prescription entity = prescriptionRepo.findById(prescriptionId)
                .orElseThrow();
        if (entity.getPharmacistStatus() != 3) return;
        // 加入队列
        auditQueue.offer(prescriptionId);
        // 查找空闲药师
        List<Pharmacist> available = pharmacistRepo
                .findByStatusAndIsOnline(1, true);
        for (Pharmacist pharmacist : available) {
            Long pid = auditQueue.poll();
            if (pid != null) {
                // 分配给药师处理
            }
        }
    }
}
```

## 4 模块交互关系

智能药师咨询模块和处方管理模块之间存在业务交互，主要体现在以下场景：

**场景1：AI咨询中涉及处方药推荐**

当用户在AI咨询中询问处方药相关问题时，AI服务会识别到处方药内容，通过风险分级将回复标记为MEDIUM或HIGH级别。此时ConsultationService会触发转人工流程，用户需要在人工药师指导下上传处方并经过处方审核流程后，才能购买处方药。

**场景2：处方审核通过后发起购药**

当处方通过药师审核后（PrescriptionService.audit方法中status=通过），系统可以自动为用户创建购物车项或推荐关联药品，此时会调用订单服务的接口。处方审核状态的变化也会通过消息中心通知用户。
