## 1 概述


### 1.1 接口设计规范

本文档定义线上购药系统（OPS）前后端交互的RESTful API接口规范。接口基于HTTP/HTTPS协议，采用JSON作为数据交换格式。

**表 1-1 接口设计规范**

| 规范项 | **说明** |
|------|------|
| 协议 | HTTPS |
| 数据格式 | JSON（application/json） |
| 编码 | UTF-8 |
| 接口前缀 | /api/v1 |
| 认证方式 | Bearer Token（JWT） |
| 请求方式 | GET（查询）、POST（创建）、PUT（更新）、DELETE（删除） |
| 时间格式 | ISO 8601（yyyy-MM-ddTHH:mm:ss） |
| 金额格式 | Decimal，单位元，保留两位小数 |

### 1.2 通用返回格式

所有接口统一返回以下格式的JSON响应：

通用响应结构

```json
{
  "code": 200,
  "message": "success",
  "data": {},
  "timestamp": 1700000000000
}
```


**表 1-2 通用响应字段**

| 字段 | 类型 | **说明** |
|------|------|------|
| code | Integer | 状态码：200成功，4xx客户端错误，5xx服务端错误 |
| message | String | 提示信息 |
| data | Object | 业务数据，接口不同返回不同 |
| timestamp | Long | 服务器响应时间戳（毫秒） |

通用错误码

400 - **请求参数**错误

401 - 未授权（Token无效或过期）

403 - 禁止访问（权限不足）

404 - 资源不存在

409 - 资源冲突（如重复提交）

429 - 请求过于频繁

500 - 服务器内部错误


## 2 药品服务接口


### 2.1 获取药品分类

`GET /api/v1/drug/categories`

**功能**：获取药品分类列表，用于首页分类导航网格展示。

**请求参数**

| 参数 | 类型 | 必填 | **说明** |
|------|------|------|------|
| parentId | Long | 否 | 父分类ID，不传则获取一级分类 |

**响应参数**（data字段）

| 参数 | 类型 | **说明** |
|------|------|------|
| id | Long | 分类ID |
| name | String | 分类名称 |
| iconUrl | String | 分类图标URL |
| children | Array | 子分类列表 |

请求示例

`GET /api/v1/drug/categories`


响应示例

```json
{
  "code": 200,
  "message": "success",
  "data": [
    { "id": 1, "name": "感冒/发烧", "iconUrl": "https://.../cold.png" },
    { "id": 2, "name": "慢性病", "iconUrl": "https://.../chronic.png" },
    { "id": 3, "name": "维生素", "iconUrl": "https://.../vitamin.png" }
  ]
}
```


### 2.2 搜索药品


`GET /api/v1/drug/search`

**功能**：按关键词搜索药品，支持模糊匹配和筛选排序。用于首页搜索栏和搜索结果页。

**请求参数**

| 参数 | 类型 | 必填 | **说明** |
|------|------|------|------|
| keyword | String | 是 | 搜索关键词（药品名/症状/拼音） |
| categoryId | Long | 否 | 分类ID筛选 |
| isRx | Integer | 否 | 是否处方药：0非处方 1处方 |
| sort | String | 否 | 排序：price_asc/price_desc/sales/rating |
| page | Integer | 否 | 页码，默认1 |
| pageSize | Integer | 否 | 每页数量，默认20 |

**响应参数**（data字段）

| 参数 | 类型 | **说明** |
|------|------|------|
| list | Array | 药品列表 |
| list[].id | Long | 药品ID |
| list[].name | String | 药品名称 |
| list[].specification | String | 规格 |
| list[].price | Decimal | 售价 |
| list[].originalPrice | Decimal | 原价 |
| list[].mainImageUrl | String | 主图URL |
| list[].isPrescription | Integer | 是否处方药：0否 1是 |
| list[].stock | Integer | 库存数量 |
| total | Long | 总记录数 |
| page | Integer | 当前页码 |

请求示例

`GET /api/v1/drug/search?keyword=感冒&sort=sales&page=1&pageSize=10`

## 3 智能咨询服务接口


### 3.1 创建咨询会话


`POST /api/v1/consult/session`

**功能**：创建AI药师咨询会话。用户进入AI咨询页时调用。

**请求头**

Authorization: Bearer {jwt_token}

Content-Type: application/json


**请求参数**

| 参数 | 类型 | 必填 | **说明** |
|------|------|------|------|
| title | String | 否 | 会话主题，默认自动生成 |

**响应参数**（data字段）

| 参数 | 类型 | **说明** |
|------|------|------|
| sessionId | Long | 会话ID |
| title | String | 会话主题 |
| createdAt | String | 创建时间 |

请求示例

`POST /api/v1/consult/session`

```json
{
  "title": "头痛用药咨询"
}
```


响应示例

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "sessionId": 50001,
    "title": "头痛用药咨询",
    "createdAt": "2025-06-11T09:30:00"
  }
}
```


### 3.2 发送咨询消息


`POST /api/v1/consult/session/{sessionId}/message`

**功能**：向AI药师发送咨询消息，获取AI回复。用于AI咨询页的对话交互。

**路径参数**

| 参数 | 类型 | 必填 | **说明** |
|------|------|------|------|
| sessionId | Long | 是 | 会话ID |

**请求体参数**

| 参数 | 类型 | 必填 | **说明** |
|------|------|------|------|
| content | String | 是 | 用户消息内容 |
| msgType | Integer | 否 | 消息类型：1文本 2语音，默认1 |

**响应参数**（data字段）

| 参数 | 类型 | **说明** |
|------|------|------|
| userMessage | Object | 用户消息对象 |
| userMessage.id | Long | 消息ID |
| userMessage.content | String | 消息内容 |
| aiReply | Object | AI回复对象 |
| aiReply.content | String | AI回复内容 |
| aiReply.symptomAnalysis | String | 症状分析 |
| aiReply.medicineAdvice | String | 用药建议 |
| aiReply.warnings | String | 注意事项/警告 |
| aiReply.riskLevel | Integer | 风险等级：1低 2中 3高 |
| aiReply.disclaimer | String | 免责声明 |
| aiReply.medicines | Array | 药品卡片列表（id/name/price/spec/stock/isRx/indications），前端渲染为可点击药品卡片 |

请求示例

`POST /api/v1/consult/session/50001/message`

```json
{
  "content": "我头痛还有点发烧，应该吃什么药？",
  "msgType": 1
}
```


响应示例

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "userMessage": {
      "id": 1001,
      "content": "我头痛还有点发烧，应该吃什么药？"
    },
    "aiReply": {
      "content": "根据您的描述，头痛伴轻微发烧可能是感冒引起的。",
      "symptomAnalysis": "头痛+低热，疑似感冒前驱症状",
      "medicineAdvice": "可考虑服用对乙酰氨基酚片（500mg）",
      "warnings": "如症状持续超过3天或高烧超过38.5℃，请及时就医",
      "riskLevel": 1,
      "disclaimer": "本建议仅供参考，请遵医嘱或咨询专业药师"
    }
  }
}
```


### 3.3 转接人工药师

`POST /api/v1/consult/session/{sessionId}/transfer`

**功能**：将当前AI咨询会话转接至人工药师。用户点击"转人工"按钮时调用。

**路径参数**

| 参数 | 类型 | 必填 | **说明** |
|------|------|------|------|
| sessionId | Long | 是 | 会话ID |

**请求体参数**

| 参数 | 类型 | 必填 | **说明** |
|------|------|------|------|
| reason | String | 否 | 转人工原因 |

**响应参数**（data字段）

| 参数 | 类型 | **说明** |
|------|------|------|
| sessionId | Long | 会话ID |
| status | String | 转接状态：waiting/connected |
| pharmacistName | String | 药师姓名（已连接时返回） |
| queuePosition | Integer | 排队位置（等待时返回） |

响应示例（排队中）

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "sessionId": 50001,
    "status": "waiting",
    "queuePosition": 2
  }
}
```


## 4 处方服务接口


### 4.1 上传处方

`POST /api/v1/prescription/upload`

**功能**：上传处方图片，系统自动触发OCR识别。用于处方上传页。

**请求头**

Authorization: Bearer {jwt_token}

Content-Type: multipart/form-data


**请求参数**

| 参数 | 类型 | 必填 | **说明** |
|------|------|------|------|
| file | File | 是 | 处方图片文件（JPEG/PNG，最大10MB） |

**响应参数**（data字段）

| 参数 | 类型 | **说明** |
|------|------|------|

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "prescriptionId": 80001,
    "imageUrl": "https://oss.com/prescription/80001.jpg",
    "ocrStatus": 1,
    "message": "处方上传成功，正在识别中..."
  }
}
```

## 8 大模型辅助接口设计（LangChain4j + RAG）

在接口设计过程中，我们利用大模型辅助完成了接口规范的制定和接口文档的生成。以下展示了提示词设计和模型反馈。

提示词设计

你是一名资深的系统架构师和API设计师，擅长RESTful API接口设计。请基于以下功能需求，为线上购药系统设计前后端交互接口。

功能需求：

1. 药品搜索和详情查看

2. AI药师咨询（创建会话、发送消息、获取回复）

3. 处方上传和状态查询

4. 购物车管理和订单创建

5. 用户登录和健康档案查询


设计要求：

1. 采用RESTful风格，接口前缀为/api/v1

2. 请求和响应数据格式为JSON

3. 每个接口包含：URI、请求方式、**请求参数**、**响应参数**、请求示例、响应示例

4. 认证方式使用JWT Bearer Token

5. 通用返回格式统一为{code, message, data, timestamp}

6. 药品相关接口不需要登录，其他接口需要登录

7. 状态码设计要合理，覆盖常见成功和错误场景


请输出完整的接口设计文档，包含所有请求和响应示例。


**大模型反馈**

模型基于功能需求生成了14个接口的完整设计：

【药品服务】3个接口
- `GET /api/v1/drug/categories` - 获取药品分类
- `GET /api/v1/drug/search` - 搜索药品（支持keyword/categoryId/isRx/sort/page/pageSize参数）
- `GET /api/v1/drug/{medicineId}` - 获取药品详情（含reviews评价列表）

【智能咨询】3个接口
- `POST /api/v1/consult/session` - 创建会话
- `POST /api/v1/consult/session/{id}/message` - 发送消息（返回结构化AI回复含symptomAnalysis/medicineAdvice/warnings/riskLevel/disclaimer）
- `POST /api/v1/consult/session/{id}/transfer` - 转接人工（返回waiting状态或connected药师信息）

【处方服务】2个接口
- `POST /api/v1/prescription/upload` - 上传处方（multipart/form-data）
- `GET /api/v1/prescription/{prescriptionId}` - 查询状态（含OCR结果和审核意见）

【订单服务】3个接口
- `POST /api/v1/cart/item` - 加入购物车
- `POST /api/v1/order` - 创建订单（含addressId/deliveryType/paymentMethod/cartItemIds）
- `GET /api/v1/order/list` - 查询订单列表（支持状态筛选和分页）

【用户服务】3个接口
- `POST /api/v1/user/login` - 登录（phone+verifyCode，返回token和userInfo）
- `GET /api/v1/user/health` - 健康档案（含allergy/chronicDiseases/currentMedications/latestBloodPressure/latestBloodGlucose/familyMembers）
- `POST /api/v1/user/reminder` - 用药提醒（medicineId/medicineName/remindTime/repeatDays）

模型特别提出了2个设计建议：

1. AI咨询接口的响应应包含结构化的字段（symptomAnalysis/medicineAdvice/warnings），便于前端渲染卡片式布局

2. 处方上传使用multipart/form-data而非JSON，因为需要传输图片文件

模型生成的接口设计经过人工评审和调整，最终形成本文档中的14个接口。主要调整包括：

- 采纳了模型建议的AI咨询结构化响应设计，前端可直接渲染分析/建议/警告卡片。
- 采纳了模型建议的处方上传使用multipart/form-data格式。
- 统一了接口前缀为/api/v1，确保版本控制。
- 补充了完整的请求和响应示例，便于前端开发人员参考。
