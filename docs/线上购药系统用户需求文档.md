### 摘要

本文档是在已完成业务需求分析的基础上，进一步确定线上购药系统用户需求的成果文件。文档系统阐述了确定业务流程的目的与意义，展示了如何运用流程图、用例图等图形工具辅助业务流程分析，以及如何利用大模型作为"业务分析助手"的角色辅助需求推导。

文档包含核心购药流程、智能咨询流程、处方审核流程、订单管理流程等关键业务流程的分析，并据此推导出涵盖药品服务、智能功能、用户管理、订单配送等维度的完整用户需求清单。每项需求均标注了优先级、来源业务流程及与业务建设目标的对应关系。

同时，文档详细展示了利用大模型辅助进行业务流程分析和用户需求推导的提示词设计、模型反馈内容以及对反馈结果的验证过程，特别是验证了用户需求与原始业务需求之间的相关性和一致性。

**关键词：用户需求；业务流程；流程图；大模型辅助；需求验证；线上购药**

## 一、概述

### 1.1 文档目的

用户需求文档是连接业务需求与系统设计的桥梁。本文档在已完成的《线上购药系统业务需求文档》基础上，通过深入分析业务流程，将抽象的业务目标转化为具体的、可执行的用户需求，为后续的系统设计和开发提供明确的功能规格说明。

文档的核心目标是回答三个关键问题：用户需要通过系统完成哪些任务？系统需要提供哪些功能来支持这些任务？这些功能如何服务于整体业务目标？

### 1.2 为什么要确定业务流程

在软件需求工程中，业务流程分析是用户需求获取的关键环节。其重要性体现在以下几个方面：

- **建立业务与系统的映射**：业务流程描述了用户完成特定目标所需要经历的一系列活动，是识别系统功能的直接来源。
- **发现隐性需求**：通过流程分析可以揭示用户在业务操作中遇到的痛点和瓶颈，从而发现用户自己可能都未明确表达的潜在需求。
- **保障需求完整性**：以流程为主线进行需求梳理，可以避免遗漏关键环节，确保需求覆盖用户使用系统的完整旅程。
- **识别系统边界**：明确哪些活动由系统支持、哪些由人工完成，以及系统与外部系统（如支付平台、物流系统）的交互接口。
- **支持后续设计**：清晰的业务流程是用户界面设计、交互设计和数据库设计的重要输入。

对于线上购药系统而言，业务流程分析尤为重要。因为医药行业的特殊性决定了其业务流程必须严格遵守法规要求，涉及处方审核、用药安全等关键环节，任何流程上的疏漏都可能带来合规风险或安全隐患。

## 二、业务流程分析

### 2.1 图形工具辅助方法

在业务流程分析中，图形化工具能够直观、清晰地展示业务活动的顺序、决策点和参与者交互。本项目采用了以下图形工具辅助业务流程分析：

**表 1 业务流程分析图形工具**

| 工具类型 | 适用场景 | 本项目应用 |
|----------|----------|-----------|
| 流程图（Flowchart） | 展示单个业务流程的步骤和决策点 | 核心购药流程、处方审核流程 |
| 跨职能流程图（Swimlane） | 展示多角色协作的业务流程 | 智能咨询流程、订单履约流程 |
| 用例图（Use Case） | 展示系统功能与参与者关系 | 系统功能模块与用户角色映射 |
| 活动图（Activity Diagram） | 展示复杂业务逻辑的并行和分支 | 推荐引擎决策流程 |

通过上述图形工具的综合运用，我们能够从不同视角全面刻画线上购药系统的业务流程，确保流程分析的深度和广度。图形化表达同时也便于与业务人员、开发团队进行沟通和确认。

### 2.2 核心业务流程梳理

基于业务需求文档的分析结果，我们梳理了线上购药系统的四大核心业务流程。以下采用流程图的形式进行展示：

**流程一：在线购药核心流程**

```
用户登录/注册 → 浏览/搜索药品 → 是否处方药？
                                    ├─ 否 → 加入购物车 → 确认订单 → 在线支付 → 订单配送 → 确认收货 → 用药服务
                                    └─ 是 → 上传处方 → 处方审核
                                                          ├─ 通过 → 加入购物车 → ...
                                                          └─ 驳回 → 通知用户修改
```

*图 1 在线购药核心流程图*

在线购药核心流程涵盖了从用户进入系统到完成购药并接收用药服务的完整链路。该流程的关键特征是区分了处方药和非处方药的处理路径，处方药必须经过上传处方和处方审核两个关键环节，体现了医药行业合规性要求。

**流程二：智能药师咨询流程**

```
用户发起咨询 → AI接收 → 症状/需求识别 → 知识库检索 → 生成建议 → 风险等级
                                                                    ├─ 低风险 → 直接回复用户 → 满意度评价
                                                                    ├─ 中风险 → 标记药师复核 → 药师审核回复 → 发送给用户 → 满意度评价
                                                                    └─ 高风险 → 转人工药师 → 人工专业解答 → 发送给用户 → 满意度评价
```

*图 2 智能药师咨询流程图*

智能药师咨询流程体现了"AI+人工"的双重保障机制。AI负责7x24小时的初步咨询和问题筛选，根据回答的风险等级决定是否需要人工药师介入。这一机制既保证了服务的及时性和可及性，又确保了用药建议的专业性和安全性。

**流程三：处方审核流程**

```
用户上传处方 → OCR识别处方 → 识别结果
                                  ├─ 清晰完整 → AI辅助解析 → 自动提取信息 → 配伍禁忌检查 → 检查结果
                                  │                                                      ├─ 无冲突 → 生成审核报告 → 药师确认 → 通过 → 通知用户购药
                                  │                                                      └─ 有冲突 → 标记异常 → 药师人工审核 → 审核结论
                                  │                                                                                      ├─ 通过 → 通知用户购药
                                  │                                                                                      └─ 驳回 → 说明原因 → 建议重新就诊
                                  └─ 模糊缺失 → 通知用户重传
```

*图 3 处方审核流程图*

处方审核流程是保障用药安全的关键环节。系统通过OCR技术自动识别处方内容，AI模型进行配伍禁忌和剂量合理性检查，最终由执业药师进行人工确认。多重审核机制确保每一张处方都经过严格把关。

**流程四：个性化推荐流程**

```
采集用户数据 → 构建用户画像 → 分析健康需求 → 匹配药品知识库 → 生成推荐列表 → 推荐类型
                                                                              ├─ 用药提醒 → 按时推送提醒
                                                                              ├─ 健康建议 → 生成科普内容
                                                                              └─ 商品推荐 → 展示相关药品
用户交互反馈 → 优化推荐模型 ↻
```

*图 4 个性化推荐流程图*

个性化推荐流程是一个持续优化的闭环系统。通过采集和分析用户的购药历史、健康档案、浏览行为等数据，系统不断精化用户画像，从而提供更加精准的用药提醒、健康建议和商品推荐服务。

### 2.3 大模型在业务流程分析中的角色

在本项目的业务流程分析中，大语言模型扮演了"业务分析助手"的核心角色。具体而言，大模型在以下几个方面提供了有效辅助：

- **流程发现助手**：通过向大模型描述业务场景和参与者，引导模型基于其知识库生成可能的业务流程路径。模型能够从行业最佳实践中提取有价值的流程设计建议，帮助分析师快速构建流程框架。
- **流程优化顾问**：在初步流程确定后，利用大模型分析流程中的瓶颈点、冗余环节和风险点。模型能够基于大量案例经验提出流程优化建议，如简化操作步骤、增加异常处理分支等。
- **合规性检查员**：针对医药行业的特殊合规要求，大模型能够协助检查流程设计是否符合药品管理法规、数据保护法规等相关规定，识别潜在的合规风险。
- **需求推导引擎**：在流程确定后，大模型能够基于流程中的每个步骤推导出对应的系统功能需求，将流程元素转化为需求条目，大幅提高需求撰写的效率。

## 三、用户需求确定

### 3.1 从业务流程到用户需求

确定业务流程后，需要将流程中的每个活动节点转化为具体的用户需求。这一转化的基本原则是：流程中的每个需要系统支持的活动，对应一个或多个用户需求；流程中的每个决策点，对应系统的判断和处理逻辑需求；流程中的每个信息流转，对应系统的数据传递和存储需求。

转化过程遵循以下步骤：首先，逐一步审视流程图中的每个节点，判断其是否需要系统支持；其次，对于需要系统支持的节点，分析用户在该节点上的目标和操作；然后，将用户目标转化为需求描述，明确**输入**、**处理**和**输出**；最后，对所有需求进行归类、去重和优先级排序。

## 四、大模型辅助用户需求分析

本章展示如何利用大模型辅助进行业务流程分析和用户需求推导，包括提示词设计、模型反馈和验证过程。

### 4.1 业务流程分析提示词

**提示词设计**

```text
你是一名资深医药电商业务分析师，擅长梳理线上购药系统的业务流程。请基于以下项目背景，帮助我完成业务流程分析。

项目背景：
```
